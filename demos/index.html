Learning

Use the following examples to learn how to use the Big Block classes and develop your own Big Block creations. If you want to quickly view some pixel animations, jump to the particle examples.

All blocks (ie: big pixels) are rendered inside a top-level <div> that acts as a viewport. The default settings are meant for rendering on iOS devices but can be adapted to other mobile devices or desktop browsers. Without changing anything, your viewport will be 320x368 pixels with each block at 8x8 pixels (40 rows, 46 cols, 1840 total blocks).

Bare bones

The easiest way to get up and running with Big Block is to point to the latest release in a <script> tag in the head of your html file. Next, in another <script> block, use the following line to create a block.

BigBlock.BlockSmall.create();

Finally, in your <body> tag, add the following onLoad event:

onload="setTimeout('BigBlock.ready()', 0)"

Defaults

The Timer class does the heavy lifting of setting up the viewport. You can override the defaults to change the viewport's background color, draw borders or alter the input feedback.

Every Big Block viewport starts to play via the following method:

BigBlock.Timer.play()

Before play() actualy does anything, it loops thru and sets any passed parameters. You can find the Timer class and a list of available parameters on the documentation page.

Fonts: characters

The Big Block framework includes three text rendering classes called Character, CharPresets and Word. Character is used to render single characters defined in CharPresets. CharPresets is not automatically instantiated when Big Block loads because not all Big Block pages include text. When you want to render text, you must include the following line:

BigBlock.CharPresets = BigBlock.CharPresets.install();

Color

The Color class loads up predefined colors during Timer.play() and can accept custom palettes via the Color.add() method. There are several predefined colors as well as predefined gradients. View the Color class on the docs page for a full list.

All color values are converted to CSS rules and should be organized by class name. To add custom color palettes, the parameter passed to the Color.add() method should be a JSON object containing all the palettes needed for the page:

var my_palette = {'classes' : [				
	{name: 'palette1',val: ["rgb(191, 192, 194);rgb(0, 0, 0);rgb(153, 153, 153)"]},
	{name: 'palette2',val: ["rgb(238, 137, 151);rgb(102, 102, 102);rgb(255, 255, 255)"]},
	{name: 'palette3',val: ["rgb(102, 102, 102);rgb(204, 204, 204);rgb(128, 145, 173)"]}
]};
			
BigBlock.Color.add(my_palette);

Each color palette is handled as an array of color values. Even if a palette only contains one color, it must be contained in an array in the JSON object. For example if you wanted a custom purple color, you would send the following to the Color class:

var my_palette = {'classes' : [				
	{name: 'grape',val: ["rgb(139, 43, 163)"]}
]};
			
BigBlock.Color.add(my_palette);

From the example above, if a BlockSmall needed to use the custom purple color, its "color" property would be set to "grape".

Grid

When Timer.play() is called, the method creates three objects that inherit from the Grid class, GridActive, GridStatic and GridText. Before Timer.play() is called, you can change the width, height and pixel dimensions of the parent Grid object. All subsequent child grids will inherit these properties.

For example, if you wanted to format the viewport for iPad, you would use the following:

	BigBlock.Grid.setProps({
		pix_dim : 8,
		width : 1024,
		height : 768
	});

The three main divs that make up the viewport will be 1024 pixels wide and 768 pixels high.

Art: basic

Rendering pixel art makes use of the BlockAnim class. Objects created from the BlockSmall class have one block. BlockAnim objects have multiple blocks organized in sets of "frames" that can be cycled to create an animation.

The "anim" property of a BlockAnim object should be formatted as an array of JSON objects like:

my_anim = [
	{'frm':
		{
			'duration' : 1,
			'pix' : [
						{'c':'white0','i':0}
					]	
		}
	}		
];

Each 'frm' entry represents a frame in the animation. The 'frm.duration' describes how many screen renders the frame should persist. 'frm.pix' is an array of blocks. The objects that make up the 'frm.pix' array describe block's color, 'c', and its position relative to the x and y property of the BlockAnim object.

Pixel art is typically created in a graphics program then run through a script to autogenerate the 'anim' array. The script I've used is not public but will soon be released.

Audio

The Audio class adds the ability to play, pause and loop audio files in a browser. Each file must have a unique name and each must be loaded via the BigBlock.Audio.add() method before the framework will try to play it. You can also pass functions to execute before and after the file is played.

Note: While most browsers are supporting more and more HTML5 components, HTML5 audio is only supported in the latest versions of Safari, Chrome, Firefox, Opera and IE 9 beta. Browsers are also divided on which audio format to support. To be safe, it's advisable to create your audio files in .wav, .ogg and .mp3 formats and pass an array to BigBlock.Audio.format of these formats (ie. BigBlock.Audio.format = ["wav", "mp3", "ogg"];). The Audio class will check if the browser supports a format before trying to play it.

Another note: While mobile browsers (ie. Mobile Safari for iPhone and iPad, Chrome for Android) have limited support for HTML5 audio, playback is problematic. Until support becomes more stable, the framework only supports playback in desktop browsers.

In the example below, each effect has been created as .wav, .ogg and .mp3 and saved in a folder called "audio". The formats passed to BigBlock.Audio.format are listed by priority. In this example, if the browser supports both .wav and .mp3, .wav will be used because it appears ahead of .mp3 in the array. To see the status of your audio files, set BigBlock.Audio.debug = true.

You may also notice the functions passed to the play(id, before_play, after_play) method. Before a file is played, the file's ready state is checked. If the file is ready, play() will execute these functions. If the file has a problem, play() will not execute these functions.
