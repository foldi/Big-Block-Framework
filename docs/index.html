Audio

  • Most methods and descriptions are taken from the W3C HTML5 spec.
  • add(params)
  • description: Adds an audio element to the DOM. Also runs load() to set up the audio file for playback.
  • params: A set of key/value pairs. "id" and "path" are required.
  • id [ string ]
  • required: true
  • description: A unique identifier... typically the name of the audio file.
  • path [ string ]
  • required: true
  • description: The path to the audio file.
  • loop [ boolean ]
  • default: false
  • description: Set to "true" to loop the audio.
  • after_load [ boolean ]
  • default: false
  • description: Pass a function to execute after the file is loaded and ready to play. If the file has any problems loading, this function will not execute.
  • load(params)
  • description: Causes the element to reset and start selecting and loading a new media resource from scratch.
  • params: A set of key/value pairs. "id" is required.
  • id [ string ]
  • required: true
  • description: A unique identifier... typically the name of the audio file.
  • play(params)
  • description: Plays an audio element that has been added to the playlist using add().
  • params: A set of key/value pairs. "id" is required.
  • id [ string ]
  • required: true
  • description: A unique identifier... typically the name of the audio file.
  • before_play [ function ]
  • default: false
  • description: A function to execute before the file plays. If the file is not ready, this function will not execute.
  • after_play [ function ]
  • default: false
  • description: A function to execute after the file plays. If the file is not ready, this function will not execute.
  • pause(params)
  • description: Sets the paused attribute to true, loading the media resource if necessary.
  • params: A set of key/value pairs. "id" is required.
  • id [ string ]
  • required: true
  • description: A unique identifier... typically the name of the audio file.
  • toggle [ boolean ]
  • default: false
  • description: If set to "true", the function will alternate between pausing and unpausing.
  • isPaused(params)
  • description: Returns true if playback is paused; false otherwise.
  • params: A set of key/value pairs. "id" is required.
  • id [ string ]
  • required: true
  • description: A unique identifier... typically the name of the audio file.
  • getPlayed(params)
  • description: Returns a TimeRanges object that represents the ranges of the media resource that the user agent has played.
  • params: A set of key/value pairs. "id" is required.
  • id [ string ]
  • required: true
  • description: A unique identifier... typically the name of the audio file.
  • isEnded(params)
  • description: Returns true if playback has reached the end of the media resource.
  • params: A set of key/value pairs. "id" is required.
  • id [ string ]
  • required: true
  • description: A unique identifier... typically the name of the audio file.
  • getPlaybackRate(params)
  • description: Returns the current rate playback, where 1.0 is normal speed.
  • params: A set of key/value pairs. "id" is required.
  • id [ string ]
  • required: true
  • description: A unique identifier... typically the name of the audio file.
  • getReadyState(params)
  • description: Returns a value that expresses the current state of the element with respect to rendering the current playback position, from the codes in the list below.
  • params: A set of key/value pairs. "id" is required.
  • id [ string ]
  • required: true
  • description: A unique identifier... typically the name of the audio file.
  • debug [ boolean ]
  • default: false
  • description: Set to "true" to send debugging information to the browser's console.
  • getNetworkState(params)
  • description: Returns the current state of network activity for the element, from the codes in the list below.
  • params: A set of key/value pairs. "id" is required.
  • id [ string ]
  • required: true
  • description: A unique identifier... typically the name of the audio file.
  • debug [ boolean ]
  • default: false
  • description: Set to "true" to send debugging information to the browser's console.
  • getDuration(params)
  • description: Returns the length of the media resource, in seconds, assuming that the start of the media resource is at time zero. Returns NaN if the duration isn't available. Returns Infinity for unbounded streams.
  • params: A set of key/value pairs. "id" is required.
  • id [ string ]
  • required: true
  • description: A unique identifier... typically the name of the audio file.
  • getCurrentTime(params)
  • description: Returns the current playback position, in seconds. Will throw an INVALID_STATE_ERR exception if there is no selected media resource.
  • params: A set of key/value pairs. "id" is required.
  • id [ string ]
  • required: true
  • description: A unique identifier... typically the name of the audio file.
  • setCurrentTime(params)
  • description: Sets the current playback position, in seconds. Will throw an INVALID_STATE_ERR exception if there is no selected media resource. Will throw an INDEX_SIZE_ERR exception if the given time is not within the ranges to which the user agent can seek.
  • params: A set of key/value pairs. "id" is required.
  • id [ string ]
  • required: true
  • description: A unique identifier... typically the name of the audio file.
  • getVolume(params)
  • description: Returns the current playback volume, as a number in the range 0.0 to 1.0, where 0.0 is the quietest and 1.0 the loudest.
  • params: A set of key/value pairs. "id" is required.
  • id [ string ]
  • required: true
  • description: A unique identifier... typically the name of the audio file.
  • setVolume(params)
  • description: Sets the current playback volume, as a number in the range 0.0 to 1.0, where 0.0 is the quietest and 1.0 the loudest. Throws an INDEX_SIZE_ERR if the new value is not in the range 0.0 .. 1.0.
  • params: A set of key/value pairs. "id" is required.
  • id [ string ]
  • required: true
  • description: A unique identifier... typically the name of the audio file.
  • mute(params)
  • description: Mutes all audio elements in playlist.
  • params: A set of key/value pairs. "id" is required.
  • id [ string ]
  • required: true
  • description: A unique identifier... typically the name of the audio file.
  • unmute(params)
  • description: Unmutes all audio elements in playlist.
  • params: A set of key/value pairs. "id" is required.
  • id [ string ]
  • required: true
  • description: A unique identifier... typically the name of the audio file.

BlockSmall

  • create(params)
  • description: Creates a single block.
  • params: A set of key/value pairs to configure the block. All parameters are optional.
  • alias [ string ]
  • default: BigBlock.getUniqueId()
  • description: A unique identifier. Use the alias to reference the object in BigBlock.Blocks and call methods.
  • example: BigBlock.Blocks.alias.destroy();
  • color [ string ]
  • default: empty
  • description: Sets the object's color. Can be a single color or set to cycle thru an array of values.
  • example: BigBlock.BlockSmall.create({color: "yellow"});
  • x [ number ]
  • default: 0
  • description: Horizontal location relative to the Grid location. Use with BigBlock.getTransform(vel, angle, x, y) to animate the objects in the viewport.
  • y [ number ]
  • default: 0
  • description: Vertical location relative to the Grid location. Use with BigBlock.getTransform(vel, angle, x, y) to animate the objects in the viewport.
  • vel [ number ]
  • default: 0
  • description: Set an animation velocity. Use with BigBlock.getTransform(vel, angle, x, y) to animate the objects in the viewport.
  • angle [ number ]
  • default: 0
  • description: Set an animation angle. Use with BigBlock.getTransform(vel, angle, x, y) to animate the objects in the viewport.
  • render [ number ]
  • default: -1
  • description: This value always decrements. If it reaches zero, the Block is ignored. Set to positive number to eventually hide this object. Leave it set to -1 to never hide the object.
  • life [ number ]
  • default: 0
  • description: This value always decrements. If it reaches zero, the Block is destroyed. Set to positive number to eventually destroy this object. Leave it set to 0 to never destroy the object.
  • render_static [ boolean ]
  • default: false
  • description: Set to "true" for blocks that will never move, change color or accept events. The RenderManager will only render these blocks once. This parameter is important for rendering performance and should always be set to "true" on static blocks. However, if a block needs to accept events, it must be set to "false".
  • action_render [ function ]
  • default: function () {}
  • description: Pass a function to run every screen render.
  • action_input [ function ]
  • default: false
  • description: Pass a function to run when the block is clicked or touched. BlockSmall.render_static must be "false".
  • after_destroy [ function ]
  • default: false
  • description: Pass a function to execute after this object is destroyed.
  • destroy()
  • description: Removes the object from BigBlock.Blocks.
  • example: BigBlock.Blocks.alias.destroy();
  • back to top

BlockAnim
  • create(params)
  • description: Creates an object made of multiple blocks. BlockAnim objects can be animated via their "anim" property.
  • params: A set of key/value pairs to configure the block. All parameters are optional.
  • alias [ string ]
  • default: BigBlock.getUniqueId()
  • description: A unique identifier. Use an object's alias to reference the object in BigBlock.Blocks and call methods. All alias's in the scene must be unique.
  • example: BigBlock.Blocks.alias.destroy();
  • x [ integer ]
  • default: 0
  • description: Horizontal location relative to the Grid location.
  • y [ integer ]
  • default: 0
  • description: Vertical location relative to the Grid location.
  • width [ integer ]
  • default: undefined
  • description: The object's width in pixels. Usually set when this block carries an "action_input".
  • height [ integer ]
  • default: undefined
  • description: The object's height in pixels. Usually set when this block carries an "action_input".
  • life [ integer ]
  • default: 0
  • description: The value decrements. Set to a positive number to eventually destroy this object.
  • after_destroy [ function ]
  • default: false
  • description: Pass a function to execute after this object is destroyed.
  • render_static [ boolean ]
  • default: false
  • description: Set to "true" for blocks that will never move, change color or accept events. The RenderManager will only render these blocks once. This parameter is important for rendering performance and should always be set to "true" on static blocks. However, if a block needs to accept events, it must be set to "false".
  • action_render [ function ]
  • default: false
  • description: Pass a function to run every screen render.
  • action_input [ function ]
  • default: false
  • description: Pass a function to run when the block is clicked or touched. SpriteSimple.render_static must be "false". Width and Height must also be defined and greater than 0.
  • anim [ array ]
  • default: 
    [
	{'frm':
		{
			'duration' : 1,
			'pix' : [
						{'c':'white0','i':0}
					],
			'label' : 'myLabel'	
		}
	}		
];
  • description: An array that describes all the frames and blocks within those frames. Frames can also have 'labels' that can be passed to goToAndPlay(arg) and goToAndStop(arg).
  • anim_state [ string ]
  • default: 'stop'
  • possible values: 'stop', 'play'
  • description: The animation's initial state.
  • anim_loop [ boolean ]
  • default: false
  • description: Set to true to loop through the frames described in the anim property.
  • goToAndPlay(arg)
  • description: Go to a frame in BlockAnim.anim and play the animation.
  • possible values for arg: A frame index, a frame label, 'nextFrame', 'lastFrame'
  • example: BigBlock.Blocks.alias.goToAndPlay();
  • goToAndStop(arg)
  • description: Go to a frame in BlockAnim.anim and stop the animation.
  • possible values for arg: A frame index, a frame label, 'nextFrame', 'lastFrame'
  • example: BigBlock.Blocks.alias.goToAndStop();
  • stop()
  • description: Stops cycling through frames in anim property. Also, stops any functions defined in BlockAnim.action_render.
  • example: BigBlock.Blocks.alias.stop();
  • play()
  • description: Cycles through frames in anim property. Also, executes any functions defined in BlockAnim.action_render.
  • example: BigBlock.Blocks.alias.play();
  • destroy(function)
  • description: Removes the object from BigBlock.Blocks.
  • function: A function to run after object is destroyed.
  • example: BigBlock.Blocks.alias.destroy();

ScreenEvent
  • The ScreenEvent class adds events to the viewport allowing authors to script events when users click or tap anywhere within its boundaries. Add events by adding functions to the following ScreenEvent properties.
  • Note: By default all mouseup, touchstart and click events call ScreenEvent.frameAdvance() and trigger any Button or Block actions at the event location.
  • screen_mouseup [ function ]
  • default: false
  • description: Adds a mouseup event to the viewport.
  • example: 
    BigBlock.ScreenEvent.screen_mouseup = function (evt_x, evt_y, event) {
	alert('mouse up!');
};
  • screen_mousedown [ function ]
  • default: false
  • description: Adds a mousedown event to the viewport.
  • screen_mousemove [ function ]
  • default: false
  • description: Adds a mousemove event to the viewport.
  • screen_touchstart [ function ]
  • default: false
  • description: Adds a touchstart event to the viewport.
  • screen_touchmove [ function ]
  • default: false
  • description: Adds a touchmove event to the viewport.
  • screen_touchend [ function ]
  • default: false
  • description: Adds a touchend event to the viewport.
  • screen_click [ function ]
  • default: false
  • description: Adds a click event to the viewport that accepts input from a Wii controller.
  • frameAdvance [ function ]
  • default: false
  • description: This function is called by the mouseup, touchstart and click events. They all pass the event object and the x, y coordinates of the event. This function is useful to set organize block objects into frames.
  • example: 
    
BigBlock.ScreenEvent.frameAdvance = function (event, x, y) {
							
	switch (BigBlock.ScreenEvent.current_frame) {
		
		case 1:
									
			BigBlock.SpriteSimple.create({
				x: BigBlock.Grid.width/2,
				y: BigBlock.Grid.height/2 - 32							
			});
			
		break;
		
	}

}

Timer

  • play(params)
  • description: Sets up the viewport and tells RenderManager to begin rendering.
  • params: A set of key/value pairs to configure the Timer. All parameters are optional.
  • before_play [ function ]
  • default: false
  • description: Called before play() and accepts one function as a parameter.
  • document_body_style [ object literal ]
  • default: true
  • description: Style the document body style using any valid CSS. Typically used to changed the background color.
  • event_buffer [ number ]
  • default: 200 milliseconds
  • description: Set the allowed time in milliseconds between processing user input events.
  • force_max_frame_rate [ boolean ]
  • default: true
  • description: If set to true, the browser will test how long it takes to render each frame. If the result is less than Timer.frame_rate, Timer will wait the necessary amount of milliseconds before processing the next frome. If set to false, the browser will try to render each frame as fast as possible.
  • frame_rate [ number ]
  • default: 33
  • description: The duration in milliseconds between screen renders. The default is 33 (1000/33 ~ 30) meaning the framework by default tries to maintain 30 frames per second. The framework however will NOT drop frames and will slow down as your screen renders become more complicated. Setting force_max_frame_rate = false will disregard Timer.frame_rate.
  • grid_active_styles [ object literal ]
  • default: true
  • description: Style the active grid using any valid CSS.
  • grid_static_styles [ object literal ]
  • default: true
  • description: Style the static grid using any valid CSS.
  • grid_text_styles [ object literal ]
  • default: true
  • description: Style the text grid using any valid CSS.
  • input_feedback [ boolean ]
  • default: true
  • description: When set to "true", a block appears and quickly cycles thru a series of colors wherever the user clicks or touches inside the viewport. Set to "false" to not include input feedback.
  • is_paused [ boolean ]
