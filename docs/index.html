How to use Big Block Framework

Big Block is library of JavaScript classes that work together to render pixel art and fonts in a web browser. The framework uses CSS2 and JavaScript 1.5 and is compatible with a wide range of browsers including Firefox, Safari, Chrome, Opera and IE from IE 6 on up.

All blocks (ie: big pixels) are rendered inside a top-level <div> that acts as a viewport. The default settings are meant for rendering on iOS devices but can be adapted to other mobile devices or desktop browsers. Without changing anything, your viewport will be 320x368 pixels with each block at 8x8 pixels (40 rows, 46 cols, 1840 total blocks).

The framework has been tested and is stable on iPhone, iPod Touch and iPad. Performance is very consistent from 3G models on up. The framework has also been tested on three types of Android phones and is stable. However, performance can vary greatly depending on the phone's processor.

Using the examples and docs below, you can render, animate and add events including touch events. The framework is still in an alpha phase and not "feature complete". However, it is stable and in use on the following sites:

  • thisisnotflash.com (for iPhone)
  • ipadpixels.com (for iPad)
  • ittybitty8bit.com

Please check these out for more elaborate uses of the framework.

Basic

The easiest way to get up and running with Big Block is to point to the latest release in a <script> tag in the head of your html file. Next, in another <script> block, use the following line to create a block.

BigBlock.SpriteSimple.create();

Finally, in your <body> tag, add the following onLoad event:

onload="setTimeout('BigBlock.ready()', 0)"

Defaults

The Timer class does the heavy lifting of setting up the viewport. You can override the defaults to change the viewport's background color, draw borders or alter the input feedback.

Every Big Block viewport starts to play via the following method:

BigBlock.Timer.play()

Before play() actualy does anything, it loops thru and sets any passed parameters. You can find the Timer class below and a list of available parameters.

Color

The Color class loads up predefined colors during Timer.play() and can accept custom palettes via the Color.add() method. There are several predefined colors as well as predefined gradients. View the Color class below for a full list.

All color values are converted to CSS rules and should be organized by class name. To add custom color palettes, the parameter passed to the Color.add() method should be a JSON object containing all the palettes needed for the page:

var my_palette = {'classes' : [				
	{name: 'palette1',val: ["rgb(191, 192, 194);rgb(0, 0, 0);rgb(153, 153, 153)"]},
	{name: 'palette2',val: ["rgb(238, 137, 151);rgb(102, 102, 102);rgb(255, 255, 255)"]},
	{name: 'palette3',val: ["rgb(102, 102, 102);rgb(204, 204, 204);rgb(128, 145, 173)"]}
]};
			
BigBlock.Color.add(my_palette);

Each color palette is handled as an array of color values. Even if a palette only contains one color, it must be contained in an array in the JSON object. For example if you wanted a custom purple color, you would send the following to the Color class:

var my_palette = {'classes' : [				
	{name: 'grape',val: ["rgb(139, 43, 163)"]}
]};
			
BigBlock.Color.add(my_palette);

From the example above, if a SpriteSimple needed to use the custom purple color, its className property would be set to "grape".

Grid

When Timer.play() is called, the method creates three objects that inherit from the Grid class, GridActive, GridStatic and GridText. Before Timer.play() is called, you can change the width, height and pixel dimensions of the parent Grid object. All subsequent child grids will inherit these properties.

For example, if you wanted to format the viewport for iPad, you would use the following:

	BigBlock.Grid.setProps({
		pix_dim : 8,
		width : 1024,
		height : 768
	});

The three main divs that make up the viewport will be 1024 pixels wide and 768 pixels high.

Art: basic

Rendering pixel art makes use of the SpriteAdvanced class. Objects created from the SpriteSimple class have one block. SpriteAdvance objects have multiple blocks organized in sets of "frames" that can be cycled to create an animation.

The "anim" property of a SpriteAdvanced object should be formatted as an array of JSON objects like:

my_anim = [
	{'frm':
		{
			'duration' : 1,
			'pix' : [
						{'c':'white0','i':0}
					]	
		}
	}		
];

Each 'frm' entry represents a frame in the animation. The 'frm.duration' describes how many screen renders the frame should persist. 'frm.pix' is an array of blocks. The objects that make up the 'frm.pix' array describe block's color, 'c', and its position relative to the x and y property of the SpriteAdvanced object.

Pixel art is typically created in a graphics program then run through a script to autogenerate the 'anim' array. The script I've used is not public but will soon be released.

  • possible values: "up", "down", "right", "left"
  • description: The direction the arrow should point.
  • className (string)
  • default: "grey"
  • description: Sets the color palette for the timeout message.
  • font (string)
  • default: "bigblock_bold"
  • description: Sets the font for the timeout message.
  • glass (boolean)
  • default: false
  • description: Toggles the glass effect.

Timer
  • play(params)
  • description: Sets up the viewport and tells RenderManager to begin rendering.
  • params: A set of key/value pairs to configure the Timer. All options are optional.
  • tap_timeout (boolean)
  • default: true
  • description: When set to "true", the tap timeout appears at the interval set in the TapTimeout object. Set to "false" to not include a tap timeout.
  • tap_timeout_params (params)
  • default: true
  • description: Send the TapTimeout some initial parameters. See the TapTimeout class for a list of parameters.
  • input_feedback (boolean)
  • default: true
  • description: When set to "true", a block with appear and quickly cycle thru a series of colors wherever you click or touch inside the viewport. Set to "false" to not include input feedback.
  • input_feedback_grad (text)
  • default: true
  • description: to change the default color cycle of input feedback block, set this to a string of rgb values in the order that they should cycle. The default lifetime of the input feedback block is 10. Provide 10 values if you want the block to change color at every screen render.
  • example: 
    "rgb(255, 255, 255);rgb(239, 239, 239);rgb(216, 216, 216);rgb(189, 189, 189);rgb(159, 159, 159);rgb(127, 127, 127);rgb(96, 96, 96);rgb(66, 66, 66);rgb(39, 39, 39);rgb(16, 16, 16)">
  • grid_text_styles (params)
  • default: true
  • description: Style the text grid using any valid CSS.
  • grid_active_styles (params)
  • default: true
  • description: Style the active grid using any valid CSS.
  • grid_static_styles (params)
  • default: true
  • description: Style the static grid using any valid CSS.
  • document_body_style (params)
  • default: true
  • description: Style the document body style using any valid CSS. Typically used to changed the background color.
  • frame_rate (integer)
  • default: true
  • description: The duration in milliseconds between screen renders. The default is 33 (1000/33 ~ 30) meaning the framework by default tries to maintain 30 frames per second. The framework however will NOT drop frames and will slow down as your screen renders become more complicated.
  • report_fr (boolean)
  • default: true
  • description: When set to "true", the browser console will output the average length in milliseconds to render the screen over the test interval (fr_rate_test_interval). Use to check if your average is greater than the frame rate set in BigBlock.Timer.frame_rate
  • fr_rate_test_interval (integer)
  • default: true
  • description: Sets the duration of the test interval.
  • before_play(function)
  • description: Called before play() and accepts a function as a parameter.
  • function: A function to run before viewport begins to render.

WordSimple
  • create(options)
  • description: Creates a word.
  • options: A set of key/value pairs to configure the word. All options are optional.
  • word_id (string)
  • default: 'word1'
  • description: A unique identifier. Use BigBlock.getObjIdByWordId(word_id) to reference the object and call methods.
  • example: BigBlock.Sprites[BigBlock.getObjIdByWordId('word1')].remove();
  • className
  • default: empty
  • description: Sets the color palette. Can be a single color or set to cycle thru an array of values. Use in combination with pulse_rate to set the cycle speed.
  • x (integer)
  • default: BigBlock.Grid.width/2
  • description: Horizontal location relative to the Grid location.
  • y (integer)
  • default: BigBlock.Grid.height/2
  • description: Vertical location relative to the Grid location.
  • value (string)
  • default: 'BIG BLOCK'
  • description: The word to render.
  • render (integer)
  • default: -1
  • description: value decrements; set to positive number to eventually hide this object.
  • pulse_rate (integer)
  • default: 10
  • description: If the CharSimple's className has more than one color, this value controls how fast the CharSimple will pulse thru the available colors in the class.
  • action (function)
  • default: function () {}
  • description: Pass a function to run every screen render.
  • afterDie (function)
  • default: null
  • description: pass a function to execute after this object is destroyed.
  • url (string or function)
  • default: false
  • description: pass a url to open or a function to execute after this object is clicked or touched.
  • link_color (color)
  • default: false
  • possible values: hex code, rgb value or plain text
  • description: Pass a hex code reference to a color. The color will appear behind the word.
  • example: The following will all create a red background behind the word 'click': 
    		
BigBlock.WordSimple.create({
	x: 0,
	value: "click",
	className: "white",
	link_color: "#ff0000"
});

BigBlock.WordSimple.create({
	x: 0,
	value: "click",
	className: "white",
	link_color: "rgb(255,0,0)"
});

BigBlock.WordSimple.create({
	x: 0,
	value: "click",
	className: "white",
	link_color: "red"
});
  • center (boolean)
  • default: false
  • description: set to true to center the word in the viewport.
  • stop()
  • description: Stops any functions defined in CharSimple.action.
  • example: BigBlock.Sprites[BigBlock.getObjIdByWordId('word1')].stop();
  • start()
  • description: Starts any functions defined in CharSimple.action.
  • example: BigBlock.Sprites[BigBlock.getObjIdByWordId('word1')].start();
  • remove(function)
  • description: Removes the word from the viewport and the object from BigBlock.Sprites.
  • function: A function to run after object is destroyed.
  • example: BigBlock.Sprites[BigBlock.getObjIdByWordId('word1')].remove(function () {alert('goodbye')});
